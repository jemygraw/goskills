Introduction

GoSkills is a Go package designed to parse and run Claude Skill packages. It adheres to the specifications found in the official Claude documentation, allowing you to build intelligent agents that can utilize a wide range of skills.

Installation

To use GoSkills in your project, you can install it via go get:

go get github.com/smallnest/goskills

You can also install the CLI tools using Homebrew:

brew tap smallnest/goskills
brew install goskills

Library Usage

Here is a simple example of how to parse a skill package:

package main

import (
    "fmt"
    "log"
    "github.com/smallnest/goskills"
)

func main() {
    skillDirectory := "./testdata/skills/artifacts-builder"
    skillPackage, err := goskills.ParseSkillPackage(skillDirectory)
    if err != nil {
        log.Fatalf("Failed to parse skill package: %v", err)
    }
    fmt.Printf("Successfully Parsed Skill: %s\n", skillPackage.Meta.Name)
}

CLI Tools

GoSkills provides two main command-line interfaces:

Skill Management CLI (goskills-cli)

Inspect and manage your local skills.

# List skills
./goskills-cli list ./testdata/skills

# Parse a specific skill
./goskills-cli parse ./testdata/skills/artifacts-builder

GoSkills Runner

The runner simulates the Claude skill-use workflow. It requires an OpenAI-compatible API key.

export OPENAI_API_KEY="YOUR_KEY"
./goskills run "create an algorithm that generates abstract art"

Custom Models

You can use other models like DeepSeek or Qianfan by specifying the API base and model name:

./goskills run \
  --model deepseek-v3 \
  --api-base https://api.deepseek.com/v1 \
  "your prompt here"

MCP Support

GoSkills supports the Model Context Protocol (MCP), allowing you to connect to external tools. You can configure MCP servers via a JSON configuration file.

By default, it looks for mcp.json in the current directory or ~/.claude.json.
